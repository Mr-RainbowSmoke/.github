# Contributing to Mr-Rainbowsmoke

Thanks for contributing. This org favors clean architecture, repeatable builds, and readable systems.

## Ways to Contribute
- Fix a bug
- Improve docs
- Propose an enhancement
- Add tests
- Improve developer experience (DX)

## Before You Start
1. Check existing issues/PRs to avoid duplicates.
2. Open an issue for larger changes to align on direction.
3. Keep PRs focused: one goal per PR.

## Development Standards (Default)
Unless a repository says otherwise:
- Use TypeScript when applicable
- Prefer small, composable modules
- Include tests for behavior changes
- Update docs when user-facing behavior changes

## Branching & Commits
- Branch naming: `feat/<name>`, `fix/<name>`, `chore/<name>`, `docs/<name>`
- Commit style: clear, present-tense, scoped when possible  
  Examples:
  - `feat(api): add event ingestion endpoint`
  - `fix(worker): handle missing env vars`

## Pull Requests
A good PR includes:
- What changed + why
- How to test it
- Screenshots (if UI)
- Notes on risk/rollout (if relevant)

### PR Checklist
- [ ] Lint/build passes
- [ ] Tests added/updated
- [ ] Docs updated (if needed)
- [ ] No secrets in code or logs
